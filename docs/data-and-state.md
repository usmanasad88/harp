# Data, prompts & on-disk state

[‚Üê Back to index](index.md)

Everything HARP reads from or writes to disk, outside the Python code.

## `data/` ‚Äî the RAG corpus

Plain markdown files; every `*.md` here is chunked at headings and BM25-indexed at startup for
the `search_knowledge` tool (see [Knowledge](knowledge.md)). Nothing is hardcoded to a specific
corpus ‚Äî drop files in, restart, and the model can ground answers in them. The current contents
are the expo/university material (event pages scraped with `scripts/scrape_site.py`, department
and research-group pages, programme/speakers/exhibitors, HARP's own description, etc.).
Non-markdown files (images, PDFs) sit here for reference but are **not** indexed today ‚Äî
`knowledge/indexer.py` is the reserved seam for richer ingestion.

## `prompts/` ‚Äî everything the models are told

Every persona, tool description, and injected context line, as editable markdown (see
[Configuration](configuration.md#the-prompts-system) for the loading/fallback mechanics). Edit a
file and restart to change agent behavior; deleting or emptying a file falls back to a built-in
default. The `#` title and `>` blockquote lines are authoring notes, stripped before the model
sees the text.

| File | Drives |
|---|---|
| `system_instructions.md` | Laila's persona ‚Äî the system instruction for every live conversation (both providers) |
| `filter_instructions.md` | The two-agent noise filter's persona (relay-only, `[[ignore]]` sentinel) |
| `search_knowledge_tool.md`, `web_search_tool.md`, `end_session_tool.md`, `move_around_tool.md`, `follow_tool.md`, `describe_scene_tool.md`, `search_memory_tool.md` | The corresponding tool `description` fields |
| `transcription_openai.md` | The OpenAI input-transcriber steering prompt (romanized Urdu) |
| `transcription_whisper.md` | The local wake-word Whisper `initial_prompt` (romanized Urdu, deliberately wake-word-free) |
| `identity_context.md`, `identity_context_with_notes.md` | The face-ID "you are talking to {name}" line, with/without notes |
| `wake_context_wave.md`, `wake_context_push_to_talk.md`, `wake_context_loud_sound.md`, `wake_context_wake_word.md` | The "why you just woke" context per wake path (`{level}`, `{text}` templates) |
| `memory_summarizer.md` | The conversation‚Üímemory instruction (`{facts}`, `{transcript}`; must request the JSON keys `summary`/`follow_up`/`person_facts`) |
| `context_writer.md` | The wake-briefing instruction (`{people}` + the camera frame) |

## `people/` ‚Äî enrollment (input to face-ID)

One folder per person; **the folder name is their stable person_id**. Each contains `info.yaml`
(`name` required; `role`, `notes` optional ‚Äî `notes` is model-facing free text that ends up in
the identity line and briefings) and 3‚Äì5 photos from different angles/lighting with exactly ONE
face each (group shots are skipped). `scripts/enroll_people.py` turns each photo into a face
embedding in the store; re-running replaces enrollment fields but preserves accumulated
summaries.

**Privacy**: photos and face fingerprints of real people. Everything here except the README, and
the built store, are gitignored ‚Äî and people should OK being enrolled.

## `assets/`

- `assets/status_voice/manifest.json` + `en/*.wav` ‚Äî the canned status clips (24 kHz mono WAVs)
  and the id ‚Üí lang ‚Üí file map `StatusVoice` resolves through. Regenerated by
  `scripts/generate_status_voice.py` (Kokoro TTS, separate venv); committed to the repo so a
  fresh clone can speak. Missing clips = HARP just stays silent about status.
- `assets/models/yolov8n-face-lindevs.onnx` ‚Äî the fast face detector used by the motion package
  (head tracking and follow mode). The other vision models (Whisper, MediaPipe gesture,
  InsightFace buffalo_l) are downloaded to user-level caches on first use instead.

## `.harp/` ‚Äî runtime state (gitignored)

| Path | Written by | Contents |
|---|---|---|
| `.harp/heartbeat` | the orchestrator | An empty file touched every `heartbeat.interval_seconds`; its mtime is the liveness signal for the (future) watchdog process |
| `.harp/logs/session-<ts>.jsonl` | `core/session_log.py` | One developer timeline per run: settings header, every bus event, every log record. Newest `keep_runs` kept |
| `.harp/memory/people/<id>.json` | `scripts/enroll_people.py` + the summarizer | The person store: enrollment fields + accumulated interaction summaries. Human-editable JSON; atomic writes |
| `.harp/memory/interactions/interaction-<ts>.*` | `memory/logger.py` + summarizer | Per-conversation transcripts. Suffix = lifecycle: `.part` (live) ‚Üí `.jsonl` (pending summary) ‚Üí `.jsonl.done` / `.jsonl.skipped` |
| `.harp/memory/guestbook.jsonl` | the summarizer | One entry per conversation whose participants were all unrecognized ‚Äî the memory is kept, no face is stored |
